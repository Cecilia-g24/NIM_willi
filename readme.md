# Project Robot Chats

## Data pipeline

`01_preprocess/preprocess.py` reads the raw export `data/data_raw/dialogs-1771498506071_raw.json`,
removes empty/malformed interactions, classifies each dialog's condition (A/B/Unknown),
detects its language, and produces the cleaned outputs described below.

### Cleaned JSON (machine-readable source of truth)

- `data/data_clean/dialogs_full.json`

This is generated by simply taking each retained raw dialog record (after dropping
empty/malformed ones) and adding four metadata fields on top — nothing else is touched:
`condition`, `condition_source`, `language`, `topic_main`. The original nested `messages`
list (`{role, content}`) is left completely unmodified: gesture markers like `<<smile>>`
stay as raw markers and the leading system prompt is preserved as-is. No text cleaning,
gesture conversion, or role relabeling happens at this stage — that only happens later,
when building the CSV rows below.

`dialogs.json` (project root) is an older, stale version of this file — it predates the
`condition_source`/`language`/`topic_main` fields and has the leading system message
stripped. It is still used as the default input by the downstream analysis scripts in
`02_visitor_fraction/`, `03_analysis_rules-based/`, `04_analysis_llm_based/`,
`05_prolific/`, and `06_liwc_features/`, but it can be safely regenerated/replaced by
`data/data_clean/dialogs_full.json`.

### CSV exports (human/annotation-friendly views)

`data/data_clean/` contains four CSVs, split by language (en/de) and by purpose. These are
generated *from* the cleaned JSON each time `preprocess.py` runs — they are not an
independent dataset, just a flattened, pre-formatted view of the same dialogs:

- `dialogs_en.csv`, `dialogs_de.csv` — audit-style export. One row per dialog, keeps system
  messages, includes turn counts (`n_system_turns`, `n_robot_turns`, `n_visitor_turns`), the
  raw messages re-embedded as a `messages_json` string column, and a human-readable
  `dialogue_original_text` rendering (role labels translated to "Roboter"/"Besucher" or
  "Robot"/"Visitor", gesture markers converted to readable bracketed text like
  `[robot smiles]`).
- `dialogs_for_annotation_en.csv`, `dialogs_for_annotation_de.csv` — stripped further for
  human annotators: system messages removed entirely, condition is hidden
  (`condition_hidden`) to keep annotation blind, and the dialogue is split into separate
  `robot_only_text` / `visitor_only_text` columns in addition to the combined
  `dialogue_for_annotation` text.
